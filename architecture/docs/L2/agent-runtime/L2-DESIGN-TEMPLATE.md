## 2. 功能规格

本章定义特性的能力边界：在范围内的是什么、不在范围内的是什么、每项能力的完备程度。功能规格描述的是"做什么"，不是"怎么做"。

### 2.1 能力清单

用表格列出每个功能点及其当前状态。功能点的粒度对准发布特性清单的子特性（一级 ✅/⬜ 项）。

| 能力 | 状态 | 说明 |
|------|------|------|
| <能力名称> | ✅ | <一句话描述当前实现覆盖了什么> |
| <能力名称> | ⬜ | <一句话描述未覆盖的部分及影响> |

状态标记：
- ✅ 已实现 — 代码完整 + 有 example 验证

- ⬜ 计划中 — 不在当前版本

| 排除项 | 原因 | 替代（如有） |
|--------|------|------------|

### 2.3 接口契约（Logical View）

定义特性对外暴露的核心接口和行为承诺。这是特性的"公共 API"——调用方、实现方、运维方依赖的契约。

#### 2.3.1 SPI / API 声明

列出特性定义的接口或类型，用 Java 伪代码（完整方法签名 + 语义 Javadoc），只列决定契约的核心方法。

```java
/** 一句话职责。 */
public interface Foo {
    /**
     * 方法用途。
     * @param context 贯穿链路的执行上下文
     * @return 统一结果流
     * @implSpec 实现者必须保证流在终端状态后关闭
     */
    Stream<Bar> execute(Baz context);
}
```

#### 2.3.2 数据类型

| 类型 | 关键字段 | 含义 | 约束 |
|------|---------|------|------|
| `Baz` | `tenantId`, `sessionId`, `taskId` | 贯穿链路的请求身份 | `tenantId`/`sessionId` 非空 |
| `Bar` | `type`, `outputContent()` | 统一执行结果 | `type` 决定结果如何被 A2A 层路由 |

#### 2.3.3 行为承诺

特性对外承诺的语义保证，用"必须/禁止/允许"等 RFC 2119 关键词表述：

- **必须**：<不可协商的行为>
- **禁止**：<不允许发生的场景>
- **允许**：<实现者可自行决定的行为>

## 4. 核心设计（Logical + Process View）

本章是文档的主体。按子特性分为 4.1、4.2 ... 小节，每个子特性覆盖关键处理流程和数据流。

### 4.x <子特性名称>

#### 4.x.1 关键处理流程

用 ASCII 时序图或流程图展示核心数据流，标注每一步的**输入→处理→输出**和**边界条件**。

```
ComponentA                    ComponentB                    ComponentC
  │                              │                              │
  │── input(data) ──────────────>│                              │
  │                              │── transform(data) → x        │
  │                              │── call(x) ──────────────────>│
  │                              │                              │── process(x) → result
  │                              │<────── result ───────────────│
  │<── output(result) ──────────│                              │
  │                              │                              │
```

流程图关键节点标注：
- **分支条件**：什么情况下走哪条路径
- **边界条件**：空输入、超时、取消的处理方式
- **状态转换**：如 Task 状态机 SUBMITTED→WORKING→COMPLETED

对于状态机，用表格形式：

| 当前状态 | 事件 | 下一状态 | 附加行为 |
|---------|------|---------|---------|
| WORKING | 结果流产出 COMPLETED | COMPLETED | emitter.complete() |
| WORKING | 结果流产出 INTERRUPTED | INPUT_REQUIRED | 保存 resume context |

## 7. 错误处理（Process View）

本章回答"出错了会怎样"——异常场景的定义和对外的行为。

| 错误场景 | 触发条件 | 行为 | 对外结果 |
|---------|---------|------|---------|
| 请求格式错误 | 请求体不是合法 JSON | 返回 JSON-RPC parse error | `{"error":{"code":-32700}}` |
| 超时 | 下游调用超过配置时限 | 记录 WARN 日志, 返回超时错误码 | `{"error":{"code":"TIMEOUT"}}` |
| 下游不可用 | 连接拒绝或 HTTP 5xx | 标记不可用, 后台重试, 返回错误 | `{"error":{"code":"UPSTREAM_UNAVAILABLE"}}` |

每个场景至少说明：谁触发、系统做什么、外面看到什么。如有降级策略，一并说明。
