# Baselines

## 目的

保存 AI-9 版本归档形成的架构基线说明，明确当前有效内容、仍为草稿的内容、已替代内容和结转开放问题。

## 适用读者

架构负责人、模块负责人、评审者、后续 AI agent。

## 维护规则

- 每个版本使用 `<version>.md` 建档。
- 基线说明必须引用对应的 version intent、architecture envelope、review packet 和 delivery projection。
- 归档时必须更新受影响 A2D 文档的 frontmatter 状态。
- 结转开放问题必须有 owner 或后续版本位置。

## 最小结构

```markdown
# <version> 架构基线

## 有效内容
## 草稿内容
## 已替代内容
## 结转开放问题
```
