# Review Packets

## 目的

保存 AI-5 生成、供 H2 审核的 4+1 架构评审包，避免评审者必须逐个阅读全部 A2D 原始产物。

## 适用读者

架构负责人、业务负责人、模块负责人、评审者。

## 维护规则

- 每个版本使用 `<version>.md` 建档。
- 评审包不得维护另一套事实，必须引用场景、能力图、模块卡、状态矩阵、ICD、不变量、harness 或验证矩阵。
- 开放问题必须有 owner 或后续位置。
- H3 前可以追加验证汇总、漂移报告、升级项和发布风险说明。

## 最小结构

```markdown
# <version> 架构审核包

## 1. 决策摘要
## 2. 4+1 视图
## 3. 变更级别汇总
## 4. 影响面
## 5. 开放问题
## 6. 验证增量
## 7. 需要人类裁决的例外
```
