# 开发指导文档结构模板

> 适用目录：`agent-runtime/docs/guides/`
> 按文档类型侧重不同章节：接口型文档重点在 §3、§5；配置型文档重点在 §2、§7；混合型两处都有内容。

---

## 1. 概述

一句话说清这个特性让开发者能做什么。紧跟一个最小示例（3-5 行代码或一条 curl），30 秒看到效果。

---

## 2. 快速开始

完整的可运行步骤。从前置条件到预期结果，开发者照做就能跑通。对应 example 项目中的最小路径。

---

## 3. 工作原理（按需）

对于概念上需要理解机制的开发者，用 ASCII 图或简短的步骤列表解释关键流程。不是设计文档的搬运——只画开发者需要知道的那一层。

```
请求 → A → B → C → 响应
         ↑
    开发者扩展点
```

---

## 4. 核心接口（接口型文档必有）

只列决定使用方式的契约点：方法签名、参数含义、返回值约定、关键约束。不列无关细节。

| 方法 | 用途 | 关键约束 |
|------|------|---------|
| `execute(ctx)` | 执行 Agent | 返回 Stream，必须由 resultAdapter() 适配 |
| `cancel(id)` | 取消执行 | OpenJiuwen 仅阻止消费，不中断 LLM |

---

## 5. 能力详述

按子特性展开，每节一个具体能力：
- 是什么（一句话）
- 怎么用（代码片段或配置片段）
- 注意事项（限制、陷阱、最佳实践）

---

## 6. 完整示例

一个端到端的工作示例，把所有能力串起来。从零开始到可运行，开发者可以复制粘贴。对应 examples 目录下最完整的那个示例项目，用代码 + 注释 + 预期输出构成。

```
// Step 1: 配置
// Step 2: 编写代码
// Step 3: 启动并验证
```

---

## 7. 配置参考（配置型文档必有）

分为两部分：
- **必配项**：不配置则特性无法工作的属性
- **可选配置**：调整行为的属性（路径 / 类型 / 默认值 / 说明）

---

## 8. 限制与已知问题

开发者选型时需要知道的约束。每个限制说明影响范围和临时方案（如有）。

---

## 9. 相关资源

- 设计文档：`architecture/docs/L2/agent-runtime/xxx.md`
- Example：`examples/xxx/`
- 其他相关开发指导

---

## 按文档类型侧重

| 类型 | 重点章节 | 代表文档 |
|------|---------|---------|
| 接口型 | §3 工作原理 → §4 核心接口 → §5 能力详述 | handler-spi、memory-services、state-persistence |
| 配置型 | §2 快速开始 → §6 完整示例 → §7 配置参考 | agent-card-configuration、remote-invocation、a2a-endpoints |
| 混合型 | §4 核心接口 + §7 配置参考 都有内容 | versatile-adapter、openjiuwen-adapter |
